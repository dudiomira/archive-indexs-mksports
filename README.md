# archive-indexs-mksports

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态形式保存，便于长期访问和索引查阅。仓库内容不针对任何特定域名或网站，仅为历史页面存档用途。

## 目录说明

```
archive-indexs-mksports/
├── index.html        # 归档总索引页
├── pages/            # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/           # 页面依赖的静态资源（可选）
    ├── css/
    ├── js/
    └── images/
```

- `index.html`：提供所有归档页面的导航入口。
- `pages/`：每个子页面独立保存，文件名保持原始命名或按日期重命名。
- `assets/`：共享或页面内引用的样式、脚本、图片等资源文件。

## 页面归档说明

- 所有归档页面均为独立完整的 HTML 文件，部分页面可能附带样式和脚本资源。
- 归档内容保留原始结构和样式，但可能移除外部链接、跟踪代码或动态元素。
- 页面标题、编码格式、基本元信息均予以保留，以确保可读性和兼容性。
- 本仓库不会对归档页面内容进行实质性修改或更新。

## 维护说明

- 本仓库不定期更新，新增的页面将通过 pull request 或直接推送的方式合并。
- 归档页面不会主动删除，但可能因存储空间或结构优化而调整目录。
- 如发现页面无法正常显示或资源缺失，欢迎提交 issue 反馈。
- 不接受对归档页面内容的修改请求，仅接受索引或资源修复相关建议。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接打开 `index.html` 即可浏览归档索引。
3. 也可通过 GitHub Pages 或其他静态托管服务部署后在线访问。
